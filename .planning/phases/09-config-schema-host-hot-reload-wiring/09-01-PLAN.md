With this composed after_reload pattern:
```python
def _after_reload():
    reapply_clip()
    window.set_bg_color(config_loader.current_config.get("bg_color", "#1a1a2e"))

config_loader = ConfigLoader(str(_cfg), pm, window.compositor, after_reload=_after_reload)
config = config_loader.load()
window.set_bg_color(config.get("bg_color", "#1a1a2e"))
config_loader.apply_config(config)
```

Key details:
- `_after_reload` is defined BEFORE `ConfigLoader` construction but the closure captures `config_loader` by reference — safe because the closure only executes after `config_loader` is bound
- `reapply_clip()` is called FIRST in `_after_reload` to preserve existing HOST-04 behavior (Pitfall 2 prevention)
- `window.set_bg_color(config.get("bg_color", "#1a1a2e"))` is called AFTER `config_loader.load()` but BEFORE `config_loader.apply_config(config)` — this applies bg_color on initial load (Pitfall 1 prevention)
- The default `"#1a1a2e"` matches the hardcoded `HostWindow._bg_qcolor` default exactly (CLR-01)
- Uses `.get()` not bracket access (CLR-01)
