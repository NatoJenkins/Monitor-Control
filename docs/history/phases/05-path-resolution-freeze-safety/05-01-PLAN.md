<objective>
Fix config path resolution and console-null-guard so both entry points work from any launch context.

Purpose: Both `host/main.py` and `control_panel/__main__.py` currently use bare `"config.json"` which resolves from the process cwd. When launched via HKCU Run key (Phase 6) or as a PyInstaller .exe (Phase 7), the cwd is `C:\Windows\System32` and config will not be found. Additionally, 12 `print()` calls in the host crash under `pythonw.exe` because `sys.stdout` is `None`. Both bugs block every subsequent v1.1 phase.

Output: `shared/paths.py` with `get_config_path()`, null-guard in `host/main.py`, updated imports in both entry points, and tests proving correctness.
</objective>

    from pathlib import Path

    _PROJECT_ROOT = Path(__file__).resolve().parent.parent

        """Return the absolute path to config.json regardless of launch cwd.

        Resolves relative to this file's location (shared/paths.py -> project root).

        """
        return _PROJECT_ROOT / "config.json"
