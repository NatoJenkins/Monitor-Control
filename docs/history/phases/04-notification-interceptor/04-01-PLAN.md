<objective>
Validate WinRT notification API compatibility in a multiprocessing.Process(spawn) subprocess before building the full notification widget.

Purpose: The asyncio/WinRT/subprocess combination is the least-validated area of this phase. This spike must confirm three things before any widget code is written: (1) asyncio.run() works in a spawn subprocess with WinRT coroutines, (2) GetAccessStatus() reflects host-granted permission in a fresh subprocess, (3) polling GetNotificationsAsync() returns notification data from a subprocess. If any of these fail, the full widget build in plan 04-02 would need a different architecture.

Output: A standalone spike script (not a pytest test) that validates all three concerns, plus requirements.txt updated with WinRT package pins and the notification widget package skeleton.
</objective>

**Spike 1: Host-side RequestAccessAsync**
- In the main process, call `asyncio.run(_request_notification_access())` where `_request_notification_access` does:
  ```python
  async def _request_notification_access():
      from winrt.windows.ui.notifications.management import (
          UserNotificationListener,
          UserNotificationListenerAccessStatus,
      )
      listener = UserNotificationListener.current
      status = await listener.request_access_async()
      return status
  ```
- Print the status value (ALLOWED / DENIED / UNSPECIFIED).
- If not ALLOWED, print a warning and continue (subprocess spike can still validate asyncio.run works).

**Spike 2: Subprocess asyncio.run + GetAccessStatus**
- Define a function `_subprocess_spike(result_queue)` that runs in a `multiprocessing.Process(target=..., start_method is inherited "spawn")`:
  ```python
  def _subprocess_spike(result_queue):
      import asyncio
      from winrt.windows.ui.notifications.management import (
          UserNotificationListener,
          UserNotificationListenerAccessStatus,
      )
      results = {}

      # Test 1: GetAccessStatus (synchronous)
      listener = UserNotificationListener.current
      status = listener.get_access_status()
      results["access_status"] = str(status)
      results["access_allowed"] = (status == UserNotificationListenerAccessStatus.ALLOWED)

      # Test 2: asyncio.run with GetNotificationsAsync
      async def _fetch():
          from winrt.windows.ui.notifications import NotificationKinds
          notifs = await listener.get_notifications_async(NotificationKinds.TOAST)
          return notifs

      try:
          notifs = asyncio.run(_fetch())
          results["asyncio_run_ok"] = True
          results["notification_count"] = len(notifs) if notifs else 0

          # Test 3: Extract data from first notification (if any)
          if notifs and len(notifs) > 0:
              n = notifs[0]
              results["creation_time_type"] = type(n.creation_time).__name__
              results["creation_time_value"] = str(n.creation_time)
              try:
                  app_name = n.app_info.display_info.display_name if n.app_info else "N/A"
                  results["app_name"] = app_name
              except Exception as e:
                  results["app_name_error"] = str(e)
              # Extract text elements
              try:
                  from winrt.windows.ui.notifications import KnownNotificationBindings
                  binding = n.notification.visual.get_binding(KnownNotificationBindings.TOAST_GENERIC)
                  if binding:
                      elements = binding.get_text_elements()
                      results["text_element_count"] = len(elements) if elements else 0
                      if elements and len(elements) > 0:
                          results["title"] = elements[0].text
                  else:
                      results["binding"] = "None (no TOAST_GENERIC binding)"
              except Exception as e:
                  results["text_extract_error"] = str(e)
      except Exception as e:
          results["asyncio_run_ok"] = False
          results["asyncio_error"] = str(e)

      result_queue.put(results)
  ```
- Spawn the subprocess with `multiprocessing.Process(target=_subprocess_spike, args=(result_queue,))`, start it, join with timeout=30.
- Read results from the queue and print a summary table.
